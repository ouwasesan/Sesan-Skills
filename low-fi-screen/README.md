# low-fi-screen

Create low-fidelity website and mobile app structures for stakeholder review before moving into detailed UI design or copywriting.

`low-fi-screen` helps turn rough ideas, screenshots, notes, approved content, or existing page structures into:

- boxed ASCII wireframes
- content architecture boards
- Figma Make prompts for low-fi prototypes

## Who This Is For

### UX Designers

Use it to quickly map a homepage, feature page, dashboard, or mobile screen before designing full UI.

### Product Managers / Product People

Use it to turn rough product ideas into something structured enough for discussion with design, engineering, or stakeholders.

### Founders / Operators

Use it when you have a homepage idea, product concept, or internal tool screen in mind and want a fast structured draft before investing in polished design or copy.

## How It Works

`low-fi-screen` takes rough input and turns it into a structured low-fidelity output.

You can give it:

- a screenshot
- a rough idea
- a list of sections
- approved homepage copy
- a competitor reference
- notes from a workshop
- an existing page you want to restructure

It can return:

- a page wireframe
- a content architecture board
- a Figma Make prompt

## Modes

### Mode A: Page Wireframe

Use this for a page-like or screen-like structure.

### Mode B: Content Architecture Board

Use this for strategy, workshop review, or stakeholder alignment.

### Mode C: Figma Prompt

Use this when you want to turn a structure into a Figma Make prompt for low-fidelity visualization.

## Example Usage

```text
Use low-fi-screen in Mode A.
Create a low-fidelity homepage wireframe for a fintech app.
Include a hero, trust section, feature overview, social proof, and CTA.
```

```text
Use low-fi-screen in Mode B.
Turn this screenshot into a content architecture board.
Show numbered blocks and label each one as KEEP, MOVE, ADD, or REVIEW.
```

```text
Use low-fi-screen in Mode C.
Convert this homepage structure into a Figma Make prompt.
Keep it grayscale, low fidelity, and suitable for stakeholder review.
```
